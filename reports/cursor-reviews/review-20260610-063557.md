## Top 10 (ranked by severity)

### 1. HIGH — The budget gate still overruns for real dispatches, and silently serializes free ones
`_BudgetGate` (RONDO-366) reserves-then-settles correctly, but its cold-start probe has two production-fatal properties the instant-mock test can't see:

```273:294:src/rondo/parallel.py
            while not self._have_sample and self._inflight > 0:
                if not self._cond.wait(timeout=_GATE_PROBE_WAIT_SEC):
                    break  # -- probe wedged — fall through rather than hang forever
            est = self._estimate if self._have_sample else 0.0
            if self._spent + self._reserved + est > self.cap:
                return None
```

- `_GATE_PROBE_WAIT_SEC = 30.0` (`parallel.py:227`). Real AI dispatches routinely run minutes (the repo's own code suspects a ~30-min thinking ceiling, `anthropic_api.py:65`). When the probe takes >30s, every waiter's `wait()` times out, `est` falls back to `0.0`, and **all remaining workers admit and dispatch** → overrun ≈ `(W-1)×cost`. The MUST it's meant to satisfy (`IFS-101` r028, `STD-107:94` r018) is violated exactly in the regime that matters.
- `settle()` only records a sample `if cost > 0` (`parallel.py:291`). The **primary path — Claude `auth=max` — has `cost_usd` 0/None** (`dispatch.py:1130`), so `_have_sample` never flips, and the gate degrades to **one-task-at-a-time for the whole round**, silently, whenever a budget is set with the default 4-worker path (`config.workers=4`, `runner.py:78`).

`test_budget_toctou_cursor.py` passes because its dispatch is an instantaneous barrier with a fixed `$1.00` cost — it exercises neither the >30s nor the zero-cost regime. The test gives false confidence.

### 2. HIGH — `CircuitBreaker._save_state` crashes provider dispatch on Windows (the exact bug the campaign fixed in audit, missed here)
The reconcile path got an `ImportError` guard (`audit.py:711-715`); `_save_state` did not:

```286:325:src/rondo/retry.py
        import fcntl  # pylint: disable=import-outside-toplevel

        try:
            ...
        except (OSError, TypeError, ValueError) as exc:
            logger.debug("Circuit breaker persist failed (non-fatal): %s", exc)
```

`import fcntl` sits **before** the `try`, and `ImportError` is not in the `except`. On Windows the first breaker transition — `record_failure` tripping, `is_open` auto-closing, `reset` — raises `ImportError` straight out of an adapter's error path (`gemini.py:179`, `chat_completions.py:259`, etc.). That is a hard crash on the failure path, directly violating `STD-110:439` req 019 (MUST: "fall back to single-writer mode and emit a WARNING — NEVER … crash"). `idempotency._compact_if_needed` guards it (`idempotency.py:230`); `audit` guards it; `retry` is the one that doesn't.

### 3. MED-HIGH — Auto-reconcile is now once-per-process: the long-lived MCP server stops doing crash forensics after startup
The RONDO-371 storm fix (good for the parallel round) gates auto-reconcile on a process-global set:

```310:322:src/rondo/audit.py
    def _claim_auto_reconcile(self) -> bool:
        ...
        with _auto_reconcile_guard:
            if key in _auto_reconciled_files:
                return False
            _auto_reconciled_files.add(key)
```

The MCP server is a persistent process (`IFS-104`). It reconciles the jsonl **once**, at the first `AuditTrail` build, then never again for that file's lifetime. A dispatch that crashes mid-flight *after* startup leaves an INTENT with no OUTCOME that auto-reconcile will never touch until the server restarts — the STD-110/STD-113 stuck-intent guarantee silently lapses for the primary deployment. `test_reconcile_scalability_cursor.py` only checks a single `run_parallel` round, so it can't see this. The fix should gate per-round, not per-process-lifetime.

### 4. MED — `anthropic_api.health()` reports a dead key as healthy; the RONDO-357 honesty fix was applied to only one adapter
`chat_completions` was corrected so 401/403 → unhealthy (`chat_completions.py:325-326`). The Anthropic adapter still does the opposite:

```425:428:src/rondo/adapters/anthropic_api.py
        except urllib.error.HTTPError as exc:
            # -- 401/403 = reachable but bad key — still "up"
            return exc.code < 500
```

So `rondo doctor` shows GREEN for the Claude-API path over an expired key, then the next dispatch dies `ERR_AUTH`. For a project whose dim-10 bar is an "honest reliability scoreboard," this is a green light over broken auth — and a textbook copy-paste-drift bug (finding #6).

### 5. MED — RONDO-334's automatic re-attempt silently doubles spend on the most expensive dispatches
On a stream disconnect the Anthropic adapter fires a second full attempt:

```285:288:src/rondo/adapters/anthropic_api.py
            if result.get("disconnect_error"):
                logger.warning("-WARNING- stream disconnected — one automatic re-attempt (RONDO-334)")
                second = retry_http(_do_request, provider_name="anthropic")
                result = _best_of_disconnects(result, second)
```

This is in direct tension with `REQ-109:259` req 213 (MUST): "Expensive thinking dispatches MUST NOT be auto-retried … doubles real spend … One visible retry **by choice**, never silent spend." A max-effort run that disconnects at ~30 min already billed its output tokens; the silent `second` bills them again — and `second` is itself a whole `retry_http` cycle (up to 5 attempts), not "one re-attempt."

### 6. MED — Adapter duplication: the hotspot is real but it is NOT gemini/ollama
Refuting the stated suspicion: `gemini.py`, `chat_completions.py`, and `anthropic_api.py` triplicate the *entire* `dispatch()` skeleton and an almost-verbatim HTTPError→error-code→breaker block (`gemini.py:168-223`, `chat_completions.py:247-297`, `anthropic_api.py:351-403`). `ProviderAdapter` exists but factors none of it. `ollama.py` is the **inverse** problem — it copies nothing and therefore has **no `retry_http`, no circuit breaker, no cost** (`ollama.py:36-76`), the lone HTTP path without the reliability primitives every other adapter has. The drift bugs in findings #4 (health) and the redaction gap (gemini redacts the body, `gemini.py:193`; chat_completions doesn't, `chat_completions.py:269-274`) are precisely what copy-paste triplication produces.

### 7. MED — `_load_state` signature TOCTOU reopens the same class of bug RONDO-368 #7 fixed
The size+mtime signature is sound, but it's recorded *after* the read, via an independent `stat()`:

```436:443:src/rondo/retry.py
            raw = self._persist_path.read_text(encoding="utf-8")
            self._record_mtime()      # <-- fresh stat(), not of `raw`
            data = json.loads(raw)
```

A peer write landing between `read_text` and the `stat()` inside `_record_mtime` makes `_persist_sig` describe bytes this instance never parsed. `_maybe_reload` then sees "no change" and the peer's trip stays invisible — the very live-visibility miss the (mtime,size) pairing was added to prevent, reintroduced on the load path. `test_breaker_reload_mtime_cursor.py` pins mtime explicitly, so it can't catch this. Capture the signature *before* the read (or hash the bytes read).

### 8. MED — A worker exception of the wrong type crashes the whole round (REQ-101 req 8)
`_dispatch_worker` catches `BaseException`, refunds, and **re-raises** (`parallel.py:336-339`). The collector only catches three types:

```185:189:src/rondo/parallel.py
    try:
        return future.result()
    except (OSError, ValueError, RuntimeError) as exc:
```

A `dispatch_task_routed` that raises `KeyError`/`TypeError` (e.g. malformed routing/config) propagates out of `as_completed` and kills `run_parallel` entirely — violating the documented `REQ-101` req 8 ("Single task failure doesn't crash others"). The sequential path and `STD-110 C7` promise isolation the parallel path doesn't deliver for non-listed exceptions.

## Holistic verdict on the campaign

**Are the Cursor tests and concurrency fixes correct?** Mostly yes, and notably well-engineered:
- **breaker fail-open (#1)**, **mtime/size reload (#7)**, **key-lock ref-count eviction (#9)**, **reconcile Windows/NFS/torn (degradation)**, **dup-outcome (`stuck_after_sec` → 900 + `final_line_torn`)** — all genuinely fixed, lock discipline is clean, no new deadlocks.
- The tests assert **observable** outcomes (on-disk JSON, OUTCOME counts, retained-lock-while-held), not method names — they survive a reshaped fix. `test_key_lock_eviction`'s in-flight-retention case and `test_reconcile_degradation`'s EWOULDBLOCK-vs-ENOLCK split are textbook.

But two are quietly inadequate: the **budget test** (#1) never runs in the slow/zero-cost regime where the fix fails, and the **scalability test** (#3) can't see the once-per-process reconcile lapse. So the green suite overstates safety on exactly the two guarantees that still don't hold in production.

**Test-suite quality:** hermetic and real (they fail on the pre-fix code by construction). Good. The systemic weakness is that the *cursor* tests model concurrency with instantaneous mocks — fine for ordering/lost-update, blind to latency- and cost-dependent failures.

**Code quality:** layering and import direction are disciplined; the audit two-phase design, secret detector, and subprocess hygiene remain genuinely strong. The DRY debt is the adapter triplication (#6) and the resulting drift bugs (#4, redaction gap).

**Still-open known items (not re-scored, for honesty):** the **fail-open sanitizer** persists — a scrub exception still spools/saves the unsanitized result (`dispatch.py:811-814`); the **idempotency PIPE_BUF claim** is still false for multi-KB/MB `raw_output` (`idempotency.py:134-148`); **cross-process idempotency** remains check-then-act; and the **advisory inline dispatch** architecture (the soft center) is unchanged.

## Bottom line: 5 / 10

Engineering *rigor* is well above average — TDD, conventions locks, append-only audit, observable-contract regression tests. But "best-in-class multi-provider AI dispatch tool" is about guarantees holding under production conditions, and several MUST-level ones do not: the budget cap fails at realistic latency and silently serializes the free path (#1), the breaker crashes on Windows (#2), reconcile stops in the main daemon (#3), and health lies on the Claude path (#4). Combined with unaddressed structural items (fail-open sanitizer, advisory inline dispatch) and the fact that it is private/unpublished/unproven, it sits at the lower-middle of the bar — solid internal tooling, not yet best-in-class. The hardening campaign moved real numbers; it also demonstrated the project's recurring pattern: a fix lands for one call site (audit ImportError, chat_completions health, RONDO-368 mtime) and its twin is missed (retry ImportError, anthropic health, the load-path TOCTOU). Closing #1 and #2 is what I'd block a release on.
