## What's LEFT — ranked

### 1. HIGH — Scrub failure still persists UNSCRUBBED (STD-114 req 006 MUST)
"Loud" fixed the silence, not the leak. On a sanitize exception the result is flagged and then persisted raw to audit JSONL, result files, spool, and history:

```816:834:src/rondo/dispatch.py
    try:
        result, _sr = sanitize_task_result(result, config=None)
    except (TypeError, AttributeError) as exc:
        # -- RONDO-388 ... FAIL-OPEN BUT LOUD.
        logger.warning("-WARNING- sanitize FAILED — result for '%s' persisted UNSCRUBBED ...")
        result.metrics["sanitize_failed"] = True
        result.context_data["sanitize_error"] = f"{type(exc).__name__}: {exc}"
```

STD-114 req 006 is MUST: "Scrub BEFORE writing to audit files, result files ... morning reports." When scrubbing breaks, that guarantee inverts. Worse, the `except` is narrow: `_scrub_dict` recurses over `parsed_result` (`sanitize.py:511-526`) and a deeply nested structure raises `RecursionError` (and a bad custom pattern raises `re.error`) — **neither is caught**, so those either crash `_finalize_dispatch` or sail past with raw data. The correct shape is withhold/quarantine on scrub failure, not persist-then-warn.

### 2. HIGH — Inline/agent dispatch bypasses every hardened guarantee (architecture)
This is the unchanged soft center, and it's the single biggest weakness:

```870:871:src/rondo/mcp_dispatch.py
    if engine["engine"] in ("inline", "agent"):
        return json.dumps(engine, indent=2)
```

For inline/agent routing Rondo returns *instructions* and never executes — so the budget gate, circuit breaker, audit INTENT/OUTCOME, sanitizer, idempotency, and spool are **all skipped**. The entire RONDO-359..390 campaign hardened the HTTP-adapter and subprocess paths; the primary Cursor/MCP usage pattern can route around all of it. You can have a perfect budget gate that simply never runs.

### 3. MED-HIGH — Audit artifacts are not `0o600` (STD-110 req 012 MUST)
`save_result` (`dispatch.py:250`) and spool (mkstemp, born `0o600`) do this right. The audit subsystem does not:

```134:138:src/rondo/audit.py
    tmp_path.write_text(content, encoding=encoding)
    os.replace(str(tmp_path), str(path))
```

`atomic_write` (prompt files + `*.result.json`) and `_jsonl_path.open("a")` (`audit.py:632`) create files at the process umask (commonly `0o644`). The dir is `0o700` (`audit.py:302`) so it's same-uid-only in practice, but the file-perm MUST is unmet — and these files store prompts, which per finding #1 can be unscrubbed. (Minor cousin: `save_result` chmods *after* write — a brief `0o644` window; spool's mkstemp pattern is the model to copy.)

### 4. MED — Budget gate under-enforces on mixed-provider rounds
A single global estimate governs all admits:

```280:308:src/rondo/parallel.py

            if self._spent + self._reserved + est > self.cap:
                return None
            ...
```

If the first task to settle is the free max-auth path, `estimate` becomes `0.0` for the *whole round*. Per-task cloud routing (RONDO-342) means a budgeted round can mix free Claude with paid Gemini/OpenAI — and every paid task then admits against `est=0.0` (`0 + reserved + 0 > cap` is false), so the cap never trips. The morning's two regressions are fixed; this heterogeneous-cost hole still touches the IFS-101/STD-107 budget MUSTs. (Also `estimate` is the *last* settled cost, not a max — it drifts down after a cheap task.)

### 5. MED — Cross-process single-flight: blocking lock + unbounded lock files
```147:158:src/rondo/idempotency.py
        try:
            fcntl.flock(lock_f.fileno(), fcntl.LOCK_EX)
        except OSError as exc:
            logger.warning("Cross-process key lock flock failed (%s) — proceeding without it", exc)
        yield
```

Two problems for the long-lived MCP server: (a) the acquire is **blocking**, so a 30-minute peer thinking-dispatch stalls *every* same-key caller cross-process for the peer's full timeout — a real latency landmine on an interactive server; (b) per-key lock files are **never unlinked** (`idempotency.py:128-129`), so the lock dir grows one inode per unique prompt forever — the on-disk twin of the in-memory `_key_locks` leak that RONDO-369 just fixed with ref-counting. Bounded in memory, unbounded on disk.

### 6. LOW-MED — Reconcile RMW doesn't hold the lock on *the audit file* (STD-110 req 016 MUST)

So reconcilers serialize against each other but **not against appenders**. The torn-final-line guard (RONDO-368) makes this practically safe, but the literal invariant — one lock, the audit file — is not held.
